# Credit Risk Scoring

Проект по построению baseline-модели кредитного скоринга для оценки вероятности дефолта клиента на основе агрегированной кредитной истории.

## Задача

Исходные данные представлены в событийном формате: одному клиенту соответствует несколько кредитных записей.  
Цель проекта — преобразовать события в клиентские признаки и обучить модель бинарной классификации для предсказания дефолта (`flag`).

## Стек

- Python
- Pandas, NumPy
- Matplotlib, Seaborn
- scikit-learn
- LightGBM

## Данные

- 12 parquet-файлов с кредитной историей клиентов
- таргет: `train_target.csv`
- формат исходных данных: несколько наблюдений на одного клиента (`id`)
- временной / порядковый индекс внутри клиента: `rn`

## Этапы проекта

### 1. Первичный анализ данных
- проверена структура исходных таблиц
- подтверждено, что одному клиенту соответствует несколько кредитов
- проанализированы типы признаков, отсутствие пропусков, распределение числа кредитов на клиента
- проверена роль `rn` как порядкового индекса кредитной истории

### 2. Агрегация событийных данных до уровня клиента
Для каждого клиента были рассчитаны:

- характеристики длины истории:
  - `loans_cnt`
  - `rn_min`, `rn_max`

- агрегаты по числовым признакам:
  - `sum`, `count`, `min`, `max`, `mean`

- признаки платежной дисциплины:
  - `paym_obs`
  - `paym_bad`
  - `paym_bad_share`
  - `paym_obs_is0`

- признаки текущего состояния клиента:
  - `last__enc_*` — значения категориальных признаков в последнем наблюдении

Обработка выполнялась батчами по parquet-файлам для снижения потребления памяти.

### 3. EDA после агрегации
После построения клиентского датасета были выполнены:

- анализ баланса классов
- анализ распределений ключевых признаков
- проверка константных и почти константных признаков
- анализ корреляции признаков с таргетом
- анализ разреженности признаков
- анализ `last`-признаков
- проверка выбросов по ключевым признакам

### 4. Feature Engineering
Дополнительно были созданы признаки:

- `has_bad_paym`
- `high_util_flag`
- `util_x_bad`
- `bad_per_loan`
- `high_risk_combo`

Также были удалены константные и почти константные признаки.

### 5. Обучение моделей
Были обучены и сравнены несколько моделей:

- Logistic Regression
- HistGradientBoostingClassifier
- LightGBM

Основная метрика качества: **ROC-AUC**, так как задача имеет выраженный дисбаланс классов.

## Результаты

### Logistic Regression
- ROC-AUC: **0.7121**

### HistGradientBoosting
- ROC-AUC: **0.7412**
- улучшенная конфигурация: **0.7421**

### LightGBM
Проведено несколько экспериментов с различными гиперпараметрами.

Лучший результат:
- **ROC-AUC: 0.7494**

## Итог

Лучшая модель — **LightGBM**, обученная на агрегированных клиентских признаках.  
Финальная модель сохранена в файл `final_pipeline.pkl`, предсказания на тестовой выборке — в `test_predictions.csv`.
